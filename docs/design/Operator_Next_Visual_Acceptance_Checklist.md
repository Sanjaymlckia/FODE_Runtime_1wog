# Operator Next Visual Acceptance Checklist

Status: Owner review required before any release.

## Readability

- [ ] At 100% browser zoom, body text is readable on every active Operator Next route.
- [ ] Secondary detail text remains readable and does not drop below the operational baseline.
- [ ] Buttons and badges are readable and visually distinct.

## Label clarity

- [ ] Primary route/worklist/payment/capability labels are human-readable.
- [ ] Raw enums do not dominate the main operator surface.
- [ ] Authority/source detail remains visible secondarily where needed.

## Review workspace

- [ ] Review opens as a dedicated readable workspace, not a cramped legacy modal experience.
- [ ] Sticky identity header stays visible.
- [ ] Section navigation clearly exposes Overview, Documents, Finance, Communications, Portal, and Audit / Technical details.
- [ ] The workspace feels like one coherent vertical scroll surface.

## Route consistency

- [ ] Lifecycle Map, Applicant Action, Finance, Communications, and Roles & Capabilities share the same visual system.
- [ ] No route contains unexplained giant blank panels or microscopic helper text.
- [ ] Worklist rows remain readable without horizontal reading strain at normal desktop width.

## Browser evidence

- [ ] Required screenshots have been captured and stored with PASS/FAIL notes.
- [ ] Current Admin fallback smoke has been visually checked.

## Release gate

- [ ] Owner sign-off recorded.
- [ ] No release actions performed before owner sign-off.
